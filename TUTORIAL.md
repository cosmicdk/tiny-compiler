## 前言：为什么你应该学编译原理

编译器 = 翻译器。把一种语言翻译成另一种语言。

### 为什么要学？

1. **理解编程语言的本质**：为什么 `1 + 2 * 3` 是 7 而不是 9？编译器里的优先级表告诉了你答案。
2. **写出更好的代码**：理解编译器怎么优化，你就能写出更容易被优化的代码。
3. **调试能力飞跃**：看懂编译错误背后的原理，不再被"undefined reference"吓到。
4. **面试加分项**：手写 Lexer/Parser 是顶级公司的常见面试题。

### 编译器的五大经典阶段

```
源代码 → 词法分析(Lexer) → 语法分析(Parser) → 语义分析(Semantic)
  → 中间代码生成(IR) → 优化(Optimizer) → 目标代码生成(CodeGen)
```

### 前端 vs 后端

```
┌───────────── 前端 (语言相关) ─────────────┐
│  Lexer  →  Parser  →  Semantic Analyzer  │
└──────────────────┬───────────────────────┘
                   ▼  (IR: 中间表示)
┌───────────── 后端 (平台相关) ─────────────┐
│  Optimizer  →  Code Generator            │
└──────────────────────────────────────────┘
```

> 💡 **为什么分前后端？** M 种语言 + N 种平台 = M+N 个模块，而不是 M×N！

## Phase 2：词法分析 —— 把字符切成"单词"

### 你已经在生活中做过无数次了

> "我今天花了42块钱买了3.14斤苹果。"

大脑自动切分：我/今天/花了/42/块钱/买了/3.14/斤/苹果/。

**这就是词法分析。** 把一串字符切成有意义的 Token，贴上类型标签。

### 代码同理

```c
if (x < 100) { return x + 1; }
```
→ `[if] [(] [x] [<] [100] [)] [{] [return] [x] [+] [1] [;] [}]`

### 核心难点

- **最长匹配原则**：能读多长读多长。"apple" 不拆成 "a"+"p"+"p"+"l"+"e"
- **前瞻 (Lookahead)**：看下一个字符决定 Token 边界。`->` 是 ARROW 还是 MINUS+GT？
- **注释和空白在词法阶段就被扔掉了**

## Phase 3：语法分析 —— 从扁平的 Token 到嵌套的树

### 为什么必须是"树"？

Token 流是扁平的：`[1] [+] [2] [*] [3]`

优先级决定树的形状：

```
   (+)              vs        (*)
  /   \                      /   \
(1)   (*)                  (+)   (3)
      /  \                /   \
    (2)  (3)            (1)   (2)
 结果: 7 ✅                  结果: 9 ❌
```

AST 的树形状天然编码运算顺序——叶子先算，根最后算。

### Pratt 解析 —— 用一张优先级表代替一堆函数

**派对能量值类比**：
- `*` `/`：能量 6，强势
- `+` `-`：能量 5，一般
- `=`：能量 1，很弱

解析 `1 + 2 * 3`：`+`(能量5) 发现 `*`(能量6>5) 更强，所以让 `*` 先算。

```
优先级表（数字越大绑定越紧）：
  ASSIGN: 1 | EQ/NEQ: 3 | LT/GT: 4 | PLUS: 5 | STAR: 6 | LPAREN: 9
```

## Phase 4：语义分析 —— 编译器终于"理解"代码了

### 为什么 Lexer + Parser 不够？

```c
let y = x + 1;       // x 没声明过 → 词法/语法都通过！
let z = "hello" + 1; // 字符串+数字 → 词法/语法都通过！
```

就像："无色的绿色想法狂怒地睡觉"——语法正确，语义荒谬。

### 符号表 —— 编译器的"户口本"

```
全局 → fn outer() { let a=1;
  → fn inner() { let b=2; print(a+b); }  ← a 查 outer, b 查自己
  → print(b);  ← ❌ b 在 inner，这里查不到！
}
```

**词法作用域**：变量在哪个花括号声明，就只能在此及子花括号使用。

查找过程：从当前作用域逐层向上翻户口本。

### 类型检查

- **类型推断**：`42`→int, `3.14`→float, `"hello"`→string
- **隐式转换**：int 可自动变 float，反之不行

## Phase 5：中间表示 —— 把树"拍平"成指令序列

### 三地址码 (TAC)

AST 是树，机器要的是线性指令：

```
a + b * c  →  t1 = b * c
              t2 = a + t1
```

**每条指令最多三个操作数（两个输入+一个输出）**

### TAC 指令种类

| 指令 | 例子 |
|------|------|
| 二元运算 | `t1 = x + y` |
| 条件跳转 | `if t1 goto L1` |
| 无条件跳转 | `goto L2` |
| 函数调用 | `t1 = call f` |
| 返回 | `return t1` |
| 标签 | `L1:` |

### if 语句的 TAC

```c
if (x < 10) { y = x + 1; } else { y = 0; }
→  t1 = x < 10
   if t1 goto L1
   goto L2
 L1:  t2 = x + 1; y = t2; goto L3
 L2:  y = 0
 L3:
```

## Phase 6：代码优化 —— 让代码更短更快

### 三种核心优化

| 优化 | 原理 | 例子 |
|------|------|------|
| **常量折叠** | 编译时计算 | `3+5` → `8` |
| **代数简化** | 恒等式 | `x+0`→`x`, `x*1`→`x`, `x*0`→`0` |
| **死代码消除** | 删掉永不执行的 | `return` 后面的全部删除 |

### 为什么提前算更好？

编译时算一次 `3+5=8`，运行时直接用 `8`——省了无数次重复计算。

### 多趟扫描直到不动点

```
while 有变化:
    常量折叠() → 代数简化() → 死代码消除()
```

## Phase 7：代码生成 —— TAC → 可执行字节码

### 栈式虚拟机

```
寄存器式：mov eax,3; add eax,5
栈式：    PUSH 3; PUSH 5; ADD
```

操作数在哪是隐式的——都在栈顶。指令更短。

### 栈式运算原理

```
PUSH 3 → [3] → PUSH 5 → [3,5] → ADD → 弹出5,弹出3 → [8]
```

### 字节码指令集

```
数据：PUSH, LOAD, STORE, POP
算术：ADD, SUB, MUL, DIV, MOD
比较：EQ, NEQ, LT, GT, LTE, GTE
控制：JMP, JMP_IF, JMP_IFN, LABEL
函数：FUNC, CALL, RET, RET_VOID
内置：PRINT, HALT
```

### 翻译规则

| TAC | 字节码 |
|-----|--------|
| `%1 = %2 + %3` | `LOAD %2; LOAD %3; ADD; STORE %1` |
| `if %1 goto L` | `LOAD %1; JMP_IF L` |
| `%1 = call f` | `CALL f; STORE %1` |
| `return %1` | `LOAD %1; RET` |

## Phase 8：虚拟机 —— 让字节码真正跑起来

### 虚拟机的状态

```
PC(程序计数器) → 操作数栈 → 变量表 → 调用栈
```

### 函数调用机制

```
调用 add(3,5):
  保存返回地址+变量表 → 从栈弹参数建新帧(a=3,b=5)
  → 执行函数体 → RET恢复调用者帧 → 返回值压入调用者栈
```

### 执行循环

```python
while not halted:
    inst = bytecodes[pc]
    执行(inst)
    pc += 1
```

### TinyLang 语法

```c
fn factorial(n: int) -> int {
    if n < 2 { return 1; }
    return n * factorial(n - 1);
}
fn main() {
    let f = factorial(5);
    print(f);  // 120
}
```
