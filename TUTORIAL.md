## 前言：为什么你应该学编译原理

### 编译器是什么？

```
源代码  →  [编译器]  →  可执行程序
C/Rust  →  [翻译官]  →  机器码/字节码
```

编译器 = 翻译器。把一种语言翻译成另一种语言。

### 本教程的核心理念

> **不空谈理论，每一阶段都产出可运行的代码。**

你将用 Python 从零构建一个编译器：TinyLang → 栈式虚拟机字节码。包含一个完整的递归阶乘函数！

### 编译器的五大经典阶段

```
源程序字符流
    → 词法分析 (Lexer)    : 把字符切成"单词"(Token)
    → 语法分析 (Parser)   : Token序列 → AST（树结构）
    → 语义分析 (Semantic) : 类型检查、作用域解析
    → 中间代码生成 (IR)    : AST → 平台无关的中间表示
    → 优化 (Optimizer)     : 常量折叠、死代码消除
    → 代码生成 (CodeGen)   : IR → 字节码
    → 虚拟机 (VM)          : 执行字节码
```

## Phase 2：词法分析

**核心概念**：把一串字符切成有意义的"单词"（Token），给每个单词贴上类型标签。

**生活类比**："我今天花了42块钱买了3.14斤苹果" → 大脑自动切出"我/今天/花了/42/块钱..."

**关键难点**：最长匹配原则、前瞻 (Lookahead)、注释和空白处理

## Phase 3：语法分析

**核心概念**：把扁平的 Token 流变成嵌套的 AST 树。树的形状天然编码了运算顺序。

**Pratt 解析**：用一张优先级表代替一堆函数。每个运算符有"能量值"（优先级），能量越大越能抓住操作数。

**优先级表**：ASSIGN=1, EQ=3, LT=4, PLUS=5, STAR=6, LPAREN=9

## Phase 4：语义分析

**核心概念**：编译器终于"理解"代码了——检查变量是否声明、类型是否匹配。

**符号表**：编译器的"户口本"。每进入作用域翻开新页，声明变量=登记，使用变量=查找，离开作用域=撕掉当前页。

**类型检查**：类型推断 + 类型兼容（int 可隐式转 float）

## Phase 5：中间表示 (IR)

**核心概念**：把 AST 树"拍平"成线性指令序列——三地址码 (TAC)。每条指令最多三个操作数。

**关键映射**：
- 二元运算 → `t1 = a op b`
- if 语句 → 条件跳转 + 标签
- 函数调用 → PARAM + CALL

## Phase 6：代码优化

**三种核心优化**：
- 常量折叠：`3+5` → `8`（编译时计算）
- 代数简化：`x+0`→`x`, `x*1`→`x`, `x*0`→`0`
- 死代码消除：return 后面的指令全部删除

**策略**：多趟扫描直到不动点

## Phase 7：代码生成

**核心概念**：TAC → 栈式字节码。栈式机器不需要寄存器名，所有操作数隐式在栈顶。

**翻译规则**：
- `%1 = %2 + %3` → `LOAD %2; LOAD %3; ADD; STORE %1`
- `if %1 goto L` → `LOAD %1; JMP_IF L`
- `%1 = call f` → `CALL f; STORE %1`

## Phase 8：虚拟机

**核心概念**：执行字节码的引擎。维护操作数栈、变量表、调用栈、PC。

**执行循环**：FETCH-DECODE-EXECUTE（取指令→解码→执行）

**函数调用**：保存返回地址+变量表 → 建立新帧 → 执行函数体 → 恢复调用者帧

## 运行方式

```bash
cd tiny_compiler
python main.py run examples/hello.tl
```

预期输出：
```
🖨️  输出: 50        ← add(42, 8)
🖨️  输出: 120       ← factorial(5)
🖨️  输出: TinyLang  ← 字符串
```

## TinyLang 语法

```c
fn factorial(n: int) -> int {
    if n < 2 { return 1; }
    return n * factorial(n - 1);
}
fn main() {
    let f = factorial(5);
    print(f);  // 输出: 120
}
```
